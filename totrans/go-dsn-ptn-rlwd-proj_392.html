Objective

The main objective for the Prototype design pattern is to avoid repetitive object creation. Imagine a default object composed of dozens of fields and embedded types. We don't want to write everything needed by this type every time that we use the object, especiallyÂ if we can mess it up by creating instances with different foundations:

  • Maintain a set of objects that will be cloned to create new instances
  • Provide a default value of some type to start working on top of it
  • Free CPU of complex object initialization to take more memory resources
